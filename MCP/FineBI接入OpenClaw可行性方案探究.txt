2. 方案​
总体流程：该方案采用“指令触发-脚本执行-结果反馈”的闭环模式。CLI 负责重活（鉴权、流处理、
容错），Skill 负责轻活（意图识别、参数提取）。​
2. 功能实现​
2.1 核心CLI实现​
功能：执行具体的 FineBI 操作。可以输入finebi-cli --help 获取全部可用命令​
输入：命令行参数（如 finebi-cli get-dashboard-user-info ）。​
输出：
•
标准输出 (stdout)：供 AI 阅读的精简 JSON 或 Markdown 表格，也可以输出文件流。​
•
标准错误 (stderr)：调试日志（AI 通常会忽略）。​
•
退出码 (Exit Code)：0  表示成功，1  表示异常（触发 AI 的错误处理逻辑）。​

2.2 OpenClaw Skill​
定义：在 OpenClaw 中注册一个 Skill。​
描述词：一个用于 OpenClaw 的 FineBI 技能，可提供数据分析和可视化功能。​
技能介绍：该技能集成了 FineBI，能够通过自然语言实现数据分析、生成报告以及查询数据集等功
能。
工具列表：get-dataset-info、preview-dataset-data、export-dashboard-pdf、export-dashboard-
excel等等。​
4. 接口设计​
对外暴露接口的原则是只对外提供从FineBI中进行“读”操作的接口，禁止暴露“写”操作接口​
命令 (Command)​
说明 (Description)​
数据集​
get-publick-datasets-list​
获取公共数据的数据集列表，分页​
preview-dataset-data​
Preview dataset data with a limit of up to 100,000 rows (预览数据集数
据，最多支持10万行)​
get-dataset-info​
Get a FineBI dataset info by tableName (通过表名获取对应的数据集信
息)​
search-my-datasets​
搜索我的分析中自助数据集​
query-dataset​
根据过滤条件查询数据集​
仪表板与平台目录​
export-dashboard-excel​
Export a dashboard to Excel format (将仪表板导出为 Excel 格式)​
export-dashboard-pdf​
Export a dashboard to PDF format (将仪表板导出为 PDF 格式)​
export-dashboard-image​
Export a dashboard to image (PNG) format (将仪表板导出为 PNG 格式)​
get-dashboard-user-info​
Get current user information and their created dashboards (获取当前
用户信息及其创建的仪表板)​
search-my-dashboards​
搜索我的分析下所有仪表板​

5. 案例​
5.1 导出仪表板并生成分析报告​
用户输入：查一下finebi里面的仪表板,分析解读一下合同金额柱形图这个仪表板（你可以先导出pdf然
后再分析解读pdf），然后生成一个飞书云文档发给我。​
finebi-cli调用链：get-dataset-info -> export-dashboard-pdf -> 飞书文档skill ​
5.2 数据同步到多维表格​

用户输入：查一下fienbi里面名为“销售明细表”的数据集，把这个数据集的数据同步到飞书多维表格
里面，多维表格的路径是合同管理表>>销售明细表​
​
finebi-cli调用链：search-datasets -> get-dataset-info -> preview-dataset-
data -> 飞书多维表格skill
5.3 定时播报数据​
用户输入：创建一个定时任务，每周一上午九点查一下finebi里面名为每周数据监控仪表板的看板，看
一下上一周的数据，然后再“南京无锡共济会”群里面通报一下数据。
